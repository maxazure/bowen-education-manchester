# 管理后台预览文件

这个目录包含了管理后台界面的独立 HTML 预览文件，可以直接在浏览器中打开查看界面设计。

## 文件说明

### 1. index.html - 登录页面
独立的登录页面预览，包含完整的样式和交互功能。

**功能特性：**
- 响应式设计，支持移动端和桌面端
- 动画效果（渐进显示、浮动背景）
- 密码可见性切换
- 表单验证
- 模拟登录功能（用于演示）

**使用方法：**
1. 直接在浏览器中打开 `index.html`
2. 输入用户名：`admin`，密码：`admin` 可以"登录"进入仪表板
3. 输入其他用户名密码会显示错误提示

### 2. dashboard.html - 仪表板页面
完整的管理后台仪表板预览，展示了后台的主要布局和功能。

**功能特性：**
- 完整的后台布局（顶部导航栏 + 侧边栏 + 主内容区）
- 侧边栏折叠/展开功能
- 移动端响应式设计（带遮罩层）
- 统计卡片展示
- 快速操作面板
- 最近活动列表
- 内容分布图表
- 系统信息展示

**使用方法：**
1. 直接在浏览器中打开 `dashboard.html`
2. 点击左上角的菜单图标可以折叠/展开侧边栏
3. 在移动端（窗口宽度 < 992px）时，侧边栏会自动隐藏，点击菜单图标显示
4. 点击右上角用户头像可以看到下拉菜单

## 与实际模板的区别

预览文件与实际模板文件的主要区别：

1. **移除了 Jinja2 模板语法**
   - 实际文件: `{% extends "base.html" %}`
   - 预览文件: 完整的独立 HTML

2. **使用静态数据**
   - 实际文件: 从数据库动态加载
   - 预览文件: 使用示例数据

3. **简化了表单提交**
   - 实际文件: POST 到后端 API
   - 预览文件: 模拟验证和跳转

4. **自包含样式**
   - 实际文件: 引用外部 CSS 文件
   - 预览文件: 所有样式内嵌在 `<style>` 标签中

## 更新建议

当实际模板文件更新时，建议同步更新预览文件：

1. 复制实际模板的 HTML 结构
2. 移除 Jinja2 模板语法
3. 替换动态数据为静态示例数据
4. 确保所有 CSS 和 JavaScript 都内联或使用 CDN

## 快速启动

### 方法 1：直接打开
双击 HTML 文件在默认浏览器中打开

### 方法 2：本地服务器
如果遇到跨域问题，可以使用本地服务器：

```bash
# Python 3
cd admin/preview
python -m http.server 8000

# 访问 http://localhost:8000
```

### 方法 3：Live Server (VS Code)
1. 安装 Live Server 扩展
2. 右键点击 HTML 文件
3. 选择 "Open with Live Server"

## 注意事项

1. **CDN 依赖** - 需要互联网连接以加载 Bootstrap 和图标字体
2. **本地存储** - 侧边栏折叠状态会保存在 localStorage 中
3. **模拟登录** - 登录功能仅用于演示，用户名密码为 admin/admin
4. **链接跳转** - 除了登录->仪表板，其他链接都是占位符
